# Тренировка с Git

Этот репозиторий создан для практики основных команд Git. Ниже описаны команды, которые нужно освоить.

## Команды для изучения:

### 1. git init
**Описание:** Инициализирует новый git репозиторий в текущей директории.

**Ключи:**
- `--bare` - создает "голый" репозиторий (без рабочей директории)

**Задание:** 
- Создайте новый репозиторий в отдельной папке
- Проверьте наличие скрытой папки .git

---

### 2. git add
**Описание:** Добавляет изменения из рабочей директории в индекс (staging area).

**Ключи:**
- `git add .` - добавить все файлы
- `git add <имя_файла>` - добавить конкретный файл
- `git add -A` - добавить все изменения (включая удаленные файлы)
- `git add -p` - интерактивное добавление частей файлов

**Задание:**
- Создайте файл test.txt
- Добавьте его в staging area
- Создайте еще один файл и добавьте его отдельной командой

---

### 3. git commit
**Описание:** Фиксирует изменения из индекса в репозиторий.

**Ключи:**
- `-m "сообщение"` - сообщение коммита
- `-a` - автоматически добавить все изменения отслеживаемых файлов
- `--amend` - изменить последний коммит

**Задание:**
- Сделайте коммит с сообщением "Добавлен тестовый файл"
- Измените файл и сделайте коммит с флагом -a

---

### 4. git status
**Описание:** Показывает состояние рабочей директории и индекса.

**Ключи:**
- `-s` - сокращенный вывод
- `-b` - показать информацию о ветке

**Задание:**
- После каждого изменения файла используйте git status
- Попробуйте вариант с ключом -s

---

### 5. git log
**Описание:** Показывает историю коммитов.

**Ключи:**
- `--oneline` - краткий вывод
- `--graph` - графическое представление веток
- `-p` - показать изменения в каждом коммите
- `-n <число>` - показать последние N коммитов

**Задание:**
- Просмотрите историю коммитов разными способами
- Используйте `git log --oneline --graph`

---

### 6. git branch
**Описание:** Работа с ветками.

**Ключи:**
- `git branch` - список веток
- `git branch <имя>` - создать ветку
- `git branch -d <имя>` - удалить ветку
- `git branch -m <новое_имя>` - переименовать текущую ветку

**Задание:**
- Создайте ветку feature/new-feature
- Переключитесь на нее
- Удалите ветку после завершения работы

---

### 7. git checkout
**Описание:** Переключение между ветками и восстановление файлов.

**Ключи:**
- `git checkout <ветка>` - переключиться на ветку
- `git checkout -b <новая_ветка>` - создать и переключиться
- `git checkout <файл>` - отменить изменения в файле

**Задание:**
- Создайте и переключитесь на ветку hotfix
- Вернитесь в основную ветку
- Отмените изменения в test.txt

---

### 8. git merge
**Описание:** Объединяет изменения из разных веток.

**Ключи:**
- `--no-ff` - отключить fast-forward слияние
- `--abort` - отменить слияние при конфликте

**Задание:**
- Создайте ветку, внесите изменения
- Вернитесь в main и выполните слияние
- Создайте конфликт и разрешите его

---

### 9. git remote
**Описание:** Работа с удаленными репозиториями.

**Ключи:**
- `git remote add <имя> <url>` - добавить удаленный репозиторий
- `git remote -v` - показать все удаленные репозитории
- `git remote remove <имя>` - удалить удаленный репозиторий

**Задание:**
- Добавьте удаленный репозиторий (например, на GitHub)
- Выведите список удаленных репозиториев

---

### 10. git push
**Описание:** Отправляет изменения в удаленный репозиторий.

**Ключи:**
- `git push <удаленный> <ветка>` - отправить изменения
- `-u` - установить upstream ветку
- `--force` - принудительная отправка (осторожно!)

**Задание:**
- Отправьте ваши изменения на удаленный репозиторий
- Создайте новую ветку и отправьте ее на удаленный сервер

---

### 11. git pull
**Описание:** Забирает изменения из удаленного репозитория.

**Ключи:**
- `git pull <удаленный> <ветка>` - получить изменения
- `--rebase` - выполнить rebase вместо merge

**Задание:**
- Внесите изменения через GitHub/GitLab интерфейс
- Получите эти изменения локально

---

### 12. git clone
**Описание:** Клонирует удаленный репозиторий.

**Ключи:**
- `git clone <url>` - клонировать репозиторий
- `--branch <ветка>` - клонировать конкретную ветку
- `--depth 1` - поверхностное клонирование

**Задание:**
- Клонируйте любой открытый репозиторий с GitHub
- Проверьте его структуру

## Как работать с этим репозиторием:

1. Изучите каждую команду
2. Выполните задания для каждой команды
3. Создавайте свои тестовые файлы и ветки
4. Практикуйтесь до автоматизма!

## Полезные советы:

- Используйте `git --help` для получения справки по любой команде
- Регулярно делайте коммиты
- Пишите осмысленные сообщения коммитов
- Следите за статусом репозитория
